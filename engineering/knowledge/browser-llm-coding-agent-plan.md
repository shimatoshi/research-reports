# ブラウザLLMをコーディングエージェントとして利用する計画書

## 概要
Android端末上で、ブラウザ版LLM（Gemini等）をコーディングエージェントとして機能させるためのモジュール構成案。

## システム構成
### 実行環境
- **端末**: Android
- **必須アプリ**:
    - **Lemur Browser**: Chromium拡張機能が動作するブラウザ
    - **Termux**: ローカルサーバーの実行およびコマンド実行環境

### 構成要素
1.  **ローカルサーバー (Termux側)**
    - 拡張機能からのリクエストを受信。
    - LLMの出力から実行コマンドや書き込み内容を抽出・実行。
    - 実行結果を拡張機能へ返信。
2.  **Chromium拡張機能 (Lemur Browser側)**
    - ブラウザ版LLMのDOM操作および通信制御。
    - LLMの出力を監視し、ローカルサーバーへ送信。
    - サーバーからのレスポンスをLLMの入力欄に自動入力・送信。

## 操作パイプライン
1.  ブラウザ（Lemur）でGemini等のLLMページを開く。
2.  Termuxでローカルサーバーを起動する。
3.  Lemurで専用の拡張機能を有効化する。

## 詳細挙動

### Chromium拡張機能
- **負荷軽減**: LLMサイトアクセス時、不要なDOM要素の読み込みやスクリプトを制限し、モバイルリソースを最適化する。
- **出力検知**: LLMの回答をリアルタイムで読み取り、解析のためにローカルサーバーへ転送する。
- **入力補助**: サーバーから返ってきた実行結果（コマンド出力やエラー等）をテキストボックスに流し込むところまで行う。**送信ボタン押下はユーザー自身が行う**（TOS配慮: UIへの完全自動化アクセスを避ける）。

### ローカルサーバー
- **コマンド実行**: 拡張機能から送られてきたテキストから、実行すべきコマンドやファイル操作を検出し、Termux環境で実行する。
- **結果返却**: 標準出力、標準エラー、またはファイル変更完了通知などを拡張機能にレスポンスとして返す。

## 特徴・メリット
- **汎用性**: Geminiに限らず、ブラウザで動作するあらゆるLLM（ChatGPT, Claude等）に応用可能。
- **リソース最適化**: 拡張機能によるDOM制御で、スペックの限られたAndroid端末でも動作を安定させる。
- **ローカル連携**: Termuxを介することで、Androidローカル環境のファイルを直接操作・ビルド・テストが可能になる。
